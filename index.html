5. Web Animations API

Declarative is best for performance


@keyframes spin3d {
    0% { transform: rotateY(0deg); }
    100% { transform: rotateY(360deg); }
}
.move-me {
    animation: spin3d 5s ease-in-out infinite;
}

.another-one {
    transition-property: transform;
    transition-duration: 2s;
}
.another-one:hover {
    transform: rotateX(45deg) scale(1.2)
}
NB. Vendor prefixes omitted for simplicity

But you often need to control through JavaScript


snowFlake.style.transform = 'translate(' + snowLeft + 'px, -100%)';
// wait a frame
snowFlake.offsetWidth;
snowFlake.style.transitionProperty = 'transform';
snowFlake.style.transitionDuration = '1500ms';
snowFlake.style.transform = 'translate(' + snowLeft + 'px, ' +
        window.innerHeight + 'px)';
Example from HTML5Rocks.com

element.animate() gives best of both


snowFlake.animate([
    {transform: 'translate(' + snowLeft + 'px, -100%)'},
    {transform: 'translate(' + snowLeft + 'px, ' + window.innerHeight + 'px)'}
], 1500);
bit.ly/elementanimate

6. General Tips

Tip 1: Define animations in CSS if possible

  • Browser can optimise; create layers, run off main thread
  • Good for simple things you don't need to control frame by frame
  • E.g. Fade out animation on delete

www.html5rocks.com/en/tutorials/speed/high-performance-animations/#toc-imperative-declarative

Example


@keyframes slide-in-left1 {
    0% {
        transform: translateX(-2000px);
    }
    10% {
        transform: translateX(0);
    }
    90% {
        transform: translateX(0);
    }
    100% {
        transform: translateX(-2000px);
    }
}
NB. Vendor prefixes omitted for simplicity

Tip 2: Stick to: translate, scale, rotate, opacity

  • The 4 things a browser can "animate cheaply"
  • “Transition everything else at your own risk”
  • Translate vs top/left: http://youtu.be/-62uPWUxgcg

www.html5rocks.com/en/tutorials/speed/high-performance-animations/

Tip 3: Use requestAnimationFrame (not setInterval/setTimeout)

  • Get called when the browser is ready
  • Helps keep framerate consistent
  • Animations in background tabs get paused
  • NB. Shim may be required for older browsers

www.html5rocks.com/en/tutorials/speed/rendering/#toc-raf

Example


function animate() {

    var snowLeft = /* some kind of calculation omitted... */

    // Update transform
    snowFlake.style.transform = 'translate(' + snowLeft + 'px)';

    // Call animate again once browser's ready for next frame
    requestAnimationFrame( animate )

}

// Kick off the first time
animate();

Tip 4: For 60 FPS, you have 16ms to prepare each frame

  • Make sure your rAF callback doesn't take longer, or framerate will drop
  • Your callback gets passed a "high resolution" timestamp
  • You can also use performance.now()

www.html5rocks.com/en/tutorials/speed/rendering/#toc-raf

Tip 5: Test, measure, optimise

  • Use stats.js to measure FPS
  • Use the Dev Tools Timeline to analyse deeper

www.html5rocks.com/en/tutorials/speed/rendering/#toc-framebudget

Tip 6: Move heavy JavaScript processing into Web Workers

  • Doing lots of processing in places like onscroll handlers is a "very common cause of terrible jankiness"
  • Common in native app development to move off from main UI thread

www.html5rocks.com/en/tutorials/workers/basics/

Tip 7: ...Or chunk it down & move into rAF callback

  • Do processing when you won't interrupt animation

www.html5rocks.com/en/tutorials/speed/rendering/

Tip 8: Use a virtual DOM system like React.js

  • Only applies minimal set of changes to DOM
  • Avoid unnecessary reflows & repaints
  • Can pause updates to components until their animation completes

http://www.phpied.com/remarkable-react/

Tip 9: Progressively enhanced animations based on frame rate?

  • @neilvoss: “Polling frame delay for selective features is pretty common in video games”
  • Could use FPS to determine animation complexity?

Tip 10: When available, use the Web Animations API...
